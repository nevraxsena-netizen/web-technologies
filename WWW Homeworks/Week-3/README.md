1-Why are we using clamp() instead of media queries?
clamp() allows values (like font sizes or spacing) to scale smoothly between a minimum and maximum without needing multiple breakpoints. It reduces code complexity and creates more fluid, responsive designs.

2-Why did we use minmax() instead of fixed columns?
minmax() makes grid columns flexible by defining a minimum and maximum size. This helps the layout adapt to different screen sizes, instead of staying rigid like fixed-width columns.

3-Why is it important to implement a mobile-first website?
Mobile-first design ensures the website works well on small screens first, then scales up for larger devices. It improves performance, accessibility, and aligns with how most users browse today.

4-What happens if we remove the variables defined at the beginning?
Removing variables makes the code harder to maintain and update. You lose consistency across styles, and small changes (like colors or spacing) would need to be repeated manually throughout the code.
